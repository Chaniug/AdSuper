## 用户需求

优化 AdSuper 项目，主要运行在 GitHub Actions 上，通过自动化流程完成规则同步。

## 产品概述

AdSuper 是一个广告规则管理项目，通过 GitHub Actions 自动同步 GitHub Issues 中的广告过滤规则到规则文件中。

## 核心功能

- 提升 GitHub Actions 运行性能，减少 workflow 执行时间
- 增强规则合并的稳定性，添加重试机制和错误处理
- 保证 GitHub Issues 能够顺利读取，处理 API 速率限制和网络异常
- 简化代码结构，提取重复逻辑，统一依赖库使用
- 添加 pip 缓存优化，加速依赖安装

## 技术栈选择

- 编程语言：Python 3.11
- GitHub API 库：PyGithub（统一使用，移除 requests 依赖）
- 包管理：pip + requirements.txt
- CI/CD：GitHub Actions

## 实现方案

### 核心策略

通过添加重试机制、统一依赖库、优化 workflow 配置来提升稳定性和性能。

### 关键技术方案

#### 1. 添加 GitHub API 重试机制

- 使用 tenacity 库或自定义重试装饰器
- 处理网络超时、API 速率限制等异常
- 添加指数退避策略

#### 2. 统一使用 PyGithub

- 移除 validation.py 中的 requests 依赖
- 统一使用 PyGithub 的 API 调用方式
- 简化依赖管理

#### 3. 添加 pip 缓存优化

- 在 GitHub Actions workflow 中添加 pip 缓存配置
- 使用 actions/setup-python 的 cache 功能
- 减少依赖安装时间

#### 4. 提取重复代码

- 将规则加载逻辑提取到 scripts/utils.py
- 统一规则验证逻辑
- 减少代码重复

#### 5. 优化搜索逻辑

- 添加分页支持，处理大量 Issues
- 优化标签过滤逻辑
- 添加详细的错误日志

## 实现注意事项

- 保持向后兼容，不破坏现有功能
- 添加详细的日志记录，便于调试
- 使用环境变量控制重试次数和超时时间
- 避免在循环中频繁调用 API

## 架构设计

### 系统架构

- 保持现有架构不变
- 优化脚本间的依赖关系
- 统一错误处理和日志记录

### 数据流

GitHub Issues → PyGithub API → 规则提取 → 规则验证 → 规则合并 → 文件输出
