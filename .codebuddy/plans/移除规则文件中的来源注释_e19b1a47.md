## 用户需求

完全移除规则文件（如 adnew.txt）中自动生成的 `! 来源: issue #X` 注释行。

## 功能说明

- 当前行为：每条从 GitHub issue 同步的规则前会自动添加 `! 来源: issue #X` 的注释
- 修改后行为：规则文件只包含规则本身，不再包含来源注释
- 影响范围：仅影响新生成的规则文件，不影响规则功能

## 核心功能

- 移除规则管理器中的来源注释输出逻辑
- 保持规则文件的简洁性

## 技术栈

- Python 3.x
- 现有项目代码结构

## 实现方案

### 修改目标文件

`d:\Adsuper\AdSuper\scripts\rule_manager.py`

### 修改位置

第 132-133 行

### 当前代码

```python
for rule in rules_by_type[rule_type]:
    if rule.source and rule.source != "existing":
        f.write(f"! 来源: {rule.source}\n")
    f.write(f"{rule.content}\n")
```

### 修改后代码

```python
for rule in rules_by_type[rule_type]:
    f.write(f"{rule.content}\n")
```

### 技术说明

1. **精确修改**：只删除来源注释的输出逻辑（第 132-133 行），保留规则内容的输出
2. **影响范围**：仅影响 `_save_rules` 方法中的文件输出逻辑
3. **兼容性**：不影响规则验证、去重、排序等其他功能
4. **向后兼容**：现有规则文件格式保持不变，只是不再添加新的来源注释

### 实施细节

- 修改类型：代码删除（2行）
- 风险评估：低 - 只影响输出格式，不影响核心逻辑
- 测试建议：修改后运行同步脚本，验证输出的规则文件不再包含 `! 来源:` 注释
